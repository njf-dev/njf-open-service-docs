    OpenApi 接口

    以下接口的调用需要提供认证信息，也就是需要带上 accessToken，请先阅读 获取 access token

    测试 accessToken

    仅用于测试 accessToken 有效性，无其他效用。

    • 请求报文示例
    GET https://open.njf2016.com/api/auth/test
Authorization: Bearer NzQzZTY**********************************jlkYjE
    • 成功响应
    {
  "content": "测试农场主",
  "status": 200,
  "message": "Ok"
}
    • 返回码说明
    返回码 描述
    200 请求成功
    • 失败响应
    HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm=njf-open-access-token

    读取网关列表

    读取当前账号的智慧农业网关列表

    • 请求报文示例
    GET https://open.njf2016.com/api/intelli/concentrators?page=1&num=16
Authorization: Bearer NzQzZTY**********************************jlkYjE
    请求参数 类型 描述
    page int 分页页码（从 1 开始）
    num int 分页内条目数量（最大 100）

    Note

    如果返回的 content 条目是空或者少于 num，则说明没有更多数据了。

    • 成功响应
    {
  "content": [
    {
      "id": "123456789",
      "concentrator": "234567891",
      "createTime": "1652256002275",
      "updateTime": "1652256002275",
      "deviceType": "SCIGW",
      "lat": 23.12911,
      "lng": 113.264,
      "name": "大米基地1",
      "online": 1,
      "baud": 9600,
      "period": 5
    }
    //    ...
  ],
  "status": 200,
  "message": "Ok"
}
    字段 描述
    concentrator 网关编号，可用于查询网关下的设备
    createTime 创建时间
    updateTime 最后更新时间
    deviceType 设备类型
    lat 维度
    lng 经度
    name 用户自定义的设备名称
    online 在线状态：1 在线，0 离线
    baud 波特率
    period 数据采集周期
    • 返回码说明
    返回码 描述
    200 请求成功
    • 失败响应
    HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm=njf-open-access-token

    读取传感器列表

    读取当前账号的智慧农业网关下的传感器列表

    • 请求报文示例
    GET https://open.njf2016.com/api/intelli/devices?concentrator=866714048687551
Authorization: Bearer NzQzZTY**********************************jlkYjE
    请求参数 类型 描述
    concentrator string 网关编号
    • 成功响应
    {
  "content": [
    {
      "id": "119684d06c0c4729816961533dca5461",
      "createtime": "2022-02-16 18:32:00",
      "valid": 1,
      "concentrator": "866714048687551",
      "circuit": "soilH",
      "code": null,
      "latestData": 8.0,
      "latestTime": 1645008377202,
      "latitude": null,
      "longitude": null,
      "name": "4#土壤湿度",
      "online": 0,
      "sequence": 0,
      "type": "soilH",
      "typeName": "土壤湿度",
      "unit": "%"
    }
    //    ...
  ],
  "status": 200,
  "message": "Ok"
}
    字段 描述
    id 传感器设备编号，可用于查询设备的数据
    createtime 创建时间
    valid 1 有效设备，0 无效设备
    concentrator 设备所在网关编号
    circuit 设备类型简称
    code 原始数据转有效数据的 js 代码
    latestData 最后上报的数据
    latestTime 最后上报数据时间
    latitude 纬度
    longitude 经度
    name 用户自定义的设备名称
    online 在线状态：1 在线，0 离线
    sequence 用户自定义的设备排序位置
    type 设备类型
    typeName 设备类型名称
    unit 数据单位
    • 返回码说明
    返回码 描述
    200 请求成功
    • 失败响应
    HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm=njf-open-access-token

    读取传感器数据

    读取当前账号的智慧农业网关下的传感器的数据

    • 请求报文示例

    此示例为了阅读方便进行格式换行，实际请求不需要这样做。

    GET https://open.njf2016.com/api/intelli/data?deviceId=119684d06c0c4729816961533dca5461
    &latest=false
    &time=2022-02-16
    &startTime=2022-02-16 18:33:00&endTime=2022-02-16 19:00:00
    &page=1
    &num=3
Authorization: Bearer NzQzZTY**********************************jlkYjE
    请求参数 类型 描述
    deviceId string 传感器设备编号
    latest boolean （可选，优先级最高）读取最后一次有数据上报的当天整天数据
    time string （可选，优先级第二）指定读取某一天的数据，支持 yyyy-MM-dd
    startTime string （可选，优先级第三）按时间区间读取数据的开始时间，支持 yyyy-MM-dd 和 yyyy-MM-dd HH:mm:ss 两种格式
    endTime string （可选，优先级第三）按时间区间读取数据的结束时间，支持 yyyy-MM-dd 和 yyyy-MM-dd HH:mm:ss 两种格式
    page int 分页页码（从 1 开始）
    num int 分页内条目数量（最大 100）

    Note

    • startTime 与 endTime 需要成对出现。

    • 如果 latest、time、startTime、endTime 都没有传参的话，会返回当天整体数据。

    • 如果返回的 content 条目是空或者少于 num，则说明没有更多数据了。

    • 成功响应
    {
  "content": [
    {
      "id": "579094",
      "deviceId": "119684d06c0c4729816961533dca5461",
      "time": "2022-02-16 18:32:04",
      "data": 8.0,
      "convertData": "8"
    }
    //    ...
  ],
  "status": 200,
  "message": "Ok"
}
    字段 描述
    deviceId 数据的传感器设备编号
    time 上报或存储时间
    data 原始值
    convertData 转换后的值
    • 返回码说明
    返回码 描述
    200 请求成功
    400 时间格式错误
    • 失败响应
    HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm=njf-open-access-token
