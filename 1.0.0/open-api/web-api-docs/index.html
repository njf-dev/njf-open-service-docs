    WebApi 接口

    以下接口是开放服务平台会使用到的接口列表。

    登入 web

    OpenWeb 登录

    采用 Basic 认证，认证成功后除返回用户信息外，token 会存储到 session 中，过期时间前都有效。

    web token 有效期一周，持续使用的话，token 会自动延期。

    • 请求报文示例
    GET {{host}}/web/auth/login
Authorization: Basic YmFzZTY0KH********************3dvcmQpKQ==
    请求头 描述
    Authorization Basic 后紧跟半角空格和 userPhone:md5(password) 文本进行 base64 编码后的字符串
    • 成功响应
    {
  "content": {
    "userId": 0,
    "updalsTassy": "****",
    "userPhone": "136********",
    "userName": "测试农场主",
    "secret": {
      "id": 2,
      "createTime": "2023-01-30 11:25:28",
      "updateTime": "2023-01-30 11:25:28",
      "valid": 1,
      "appId": "2b456c*******************",
      "secret": "7e145b*******************"
    },
    "ips": "127.0.0.1,192.168.0.1,168.168.168.168"
  },
  "status": 200,
  "message": "Ok"
}
    字段名 描述
    secret 开放接口中使用到的 appId 和 secret 信息
    ips ip 列表
    • 失败响应
    HTTP/1.1 401 Unauthorized
WWW-Authenticate: Basic realm=njf-open-web, charset=UTF-8
    • 返回码说明
    返回码 描述
    200 请求成功
    401 Authorization 错误

    登出 web

    一旦登出，web token 立即失效。

    • 请求报文示例
    GET {{host}}/web/auth/logout
    • 成功响应
    {
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    扫码登录

    获取扫码登录 codeId

    扫码登录时通过此接口获取 codeId，前端使用 codeId 查询后续扫码、确认状态。

    参考 扫码登录时序图

    • 请求报文示例
    GET {{host}}/web/auth/qrcode
    • 成功响应
    {
  "content": {
    "codeId": "b6d00e61f*********************",
    "expiryTime": "2023-02-02 10:26:22"
  },
  "status": 200,
  "message": "Ok"
}
    字段 描述
    codeId 用于查询此 codeId 状态
    expiryTime 过期时间
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    查询 codeId 状态

    查询 codeId 扫码状态和确认状态。

    若用户确认登录，则后续逻辑与 /web/auth/login 一样，除返回用户信息外，token 会存储到 session 中，过期时间前都有效。

    Note

    此接口请将超时时间调整为 5 分钟，服务端会在状态变化后返回信息。

    • 请求报文示例
    GET {{host}}/web/auth/qrcode/status?codeId=b6d00e61f*********************
    &status=0
    请求参数 类型 描述
    codeId string coedId
    status int 要查询的状态：
    0 查询扫码状态 成功扫码会返回 201、
    1 查询确认状态 确认登录会返回 200 和用户信息，token 会存储到 session，取消登录会返回 203
    • 成功响应
    200
    {
  "content": {
    "userId": 0,
    "updalsTassy": "****",
    "userPhone": "136********",
    "userName": "测试农场主",
    "secret": {
      "id": 2,
      "createTime": "2023-01-30 11:25:28",
      "updateTime": "2023-01-30 11:25:28",
      "valid": 1,
      "appId": "2b456c*******************",
      "secret": "7e145b*******************"
    },
    "ips": "127.0.0.1,192.168.0.1,168.168.168.168"
  },
  "status": 200,
  "message": "Ok"
}
    201
    {
  "status": 201,
  "message": "The user has scanned the qrcode and waited for the user's confirmation"
}
    字段 描述
    concentrator 网关编号，可用于查询网关下的设备
    status codeId 的状态码：
    200 用户确认登录，成功返回信息与 /web/auth/login 一样、
    201 用户扫码成功、
    203 用户取消登录
    • 失败响应
    {
  "status": 402,
  "message": "code has expired"
}
    字段 描述
    status 401 查询超时
    402 codeId 过期
    403 参数 status 错误
    404 用户不存在或信息错误

    用户扫码和确认信息上报

    app 上报用户扫码状态和确认状态

    • 请求报文示例
    POST {{host}}/web/auth/qrcode/status
Content-Type: application/x-www-form-urlencoded
token: this_is_a_********_token_from_app

codeId=f96f3e18865f4b00a8324166cd507b8c&status=1
    请求头 描述
    Content-Type 表单提交
    token app 端的 token
    表单参数 类型 描述
    codeId string coedId
    status int 上报用户操作：
    1 用户用户扫码打开页面后即上报、
    2 用户确认登录时上报，业务结束
    3 用户取消时上报，业务结束
    • 成功响应
    {
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 401,
  "message": "please add token"
}
    字段 描述
    status 401 token 不能为空
    402 codeId 过期
    403 codeId 不能为空
    404 token 无效或过期
    405 参数 status 错误
    406 codeId 状态异常，可能是被其他用户扫过或上报过了

    业务接口

    读取 secret

    读取用户在开放接口中使用到的 appId 和 secret 信息

    • 请求报文示例
    GET {{host}}/web/secret
    • 成功响应
    {
  "content": {
    "id": 2,
    "createTime": "2023-01-30 11:25:28",
    "updateTime": "2023-01-30 11:25:28",
    "valid": 1,
    "appId": "2b456c*******************",
    "secret": "7e145b*******************"
  },
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    更新 secret

    若业务端发现 secret 泄露，可以生成新的 secret。

    Note

    系统会延长 oldSecret 有效期 10 分钟，以便业务端平稳过渡业务。

    • 请求报文示例
    POST {{host}}/web/secret/update
    • 成功响应
    {
  "content": {
    "id": 2,
    "createTime": "2023-01-30 11:25:28",
    "updateTime": "2023-02-02 10:02:02",
    "valid": 1,
    "appId": "2b456c*******************",
    "secret": "166189a******************"
  },
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    列出 ip 白名单

    列出业务端服务器的 IP 列表

    • 请求报文示例
    GET {{host}}/web/whitelist
    • 成功响应
    {
  "content": "127.0.0.1,192.168.0.1,168.168.168.168",
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    添加 ip 白名单

    业务端将 IP 加入白名单

    • 请求报文示例
    POST {{host}}/web/whitelist/127.0.0.1%2C192.168.0.1%2C168.168.168.168
    请求类型 类型 描述
    POST string 一次可添加多个 ip，用半角逗号链接起来
    • 成功响应
    {
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    删除 ips

    业务端移除 IP

    • 请求报文示例
    DELETE {{host}}/web/whitelist/192.168.0.1%2C168.168.168.168
    请求类型 类型 描述
    DELETE string 一次可移除多个 ip，用半角逗号链接起来
    • 成功响应
    {
  "status": 200,
  "message": "Ok"
}
    • 失败响应
    {
  "status": 400,
  "message": "authentication failed"
}

    业务端可设置回调接口，农卷风服务将在设备上报数据时主动推送信息到业务端服务器
