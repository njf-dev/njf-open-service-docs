    AccessToken

    农卷风开放服务使用 accessToken 作为平台全局唯一接口调用凭据，除获取 accessToken 的接口外，调用其他接口时都需带上 accessToken。

    1. 用于校验接口访问/调用是否有权限，有效期暂定为 24 小时，有效期内不需要重复申请，可以重复使用
    2. 但是一旦获取新 accessToken 会导致旧 token 失效，所以请在中控服务器统一获取和刷新 token
    3. 获取 accessToken 请先配置 IP 白名单，否则会出现接口调用异常

    Note

    获取新 token 后，旧 token 会延长 10 分钟失效，保证业务端平稳过度。 但是，如果旧 token 在刷新时刚好过期，则无法保证成功，所以请提前几分钟获取新 token。

    获取 accessToken

    为保证管理者个人信息安全，我们采用 appId 和 secret 的方式进行 token 认证，这样即保留了认证账号的简单可辨识，又保证了主账号信息安全。

    appId 和 secret 请通过 开放平台 获取

    业务端通过访问 /api/auth/token 接口获得 accessToken，访问此接口需要在请求头的 Authorization 带上 Basic 信息，Basic 的内容是 appId:secret 的 base64 编码凭据。

    • 请求报文示例
    GET https://open.njf2016.com/api/auth/token
Authorization: Basic N2FjNjZj*******************************************WVkNGUwNzY=
    请求头 描述
    Authorization Basic 后紧跟半角空格和 appId:secret 文本进行 base64 编码后的字符串
    • 成功响应
    {
  "content": {
    "accessToken": "NzQzZTY**********************************jlkYjE",
    "expiresIn": "2023-01-31 10:02:55"
  },
  "status": 200,
  "message": "Ok"
}
    字段名 描述
    accessToken 获取的 accessToken
    expiresIn 具体过期时间
    • 失败响应
    HTTP/1.1 401 Unauthorized
WWW-Authenticate: Basic realm=njf-open-service, charset=UTF-8
    • 返回码说明
    返回码 描述
    200 请求成功
    401 Authorization 错误

    使用 accessToken

    成功获得 accessToken 后，后续接口都需要在请求头的 Authorization 带上 Bearer 信息，Bearer 的内容是 /api/auth/token 接口获得的 accessToken 字符串，注意这里的内容不需要 base64 编码。

    • 请求报文示例
    GET https://open.njf2016.com/api/auth/test
Authorization: Bearer NzQzZTY**********************************jlkYjE

    上面是一个示例，具体接口请查看 OpenApi 接口文档
